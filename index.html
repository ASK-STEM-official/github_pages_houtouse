GitHub Pagesってなーに

GitHub PagesはGitHub上にあるリポジトリを参照してサイト展開できる素敵サービスだよ。 普通サイトを公開するとなると必要になるドメインを取らなくていいのもつよ味。 じゃあドメインどうなっちゃうんだよって話になるけどhttps://(アカウント名).github.io/(リポジトリ名)の形式にしてるから、リンクがちょっと長くなるデメリットがある。 でもお金がない現代高校生のWebの最初の一歩としてはいいんではなかろうかと思う。

GitHub Pagesで出来ない事

サーバーサイドの実装ができません。GitHubが考えたさいつよのサーバー処理を実現するためにサーバーサイトは書けません。localhostで我慢してね。

つかってみよう

どでかく書いたはいいものの実はGit講習の延長みたいなところにあるのでリモートリポジトリが作れてて、なおかつVSCodeなりで編集できるならもう使えるんですよね。というわけで設定は簡単

1.リモートリポジトリにindex.htmlを作る

GitHub Pagesはindex.htmlをホーム画面にしてくれるのでindex.htmlを作ってそこにトップページを書いてください。

2.GitHubのリモートリポジトリで設定をしよう

ページにしたいリポジトリを選びます。そして設定を開きます。そうすればPagesという項目があるのでBuild and deploymentのBranchをNoneからmainにしてSaveしてください。そうすればほら。あなたの素敵なサイトはページとして構成してくれますよ。
